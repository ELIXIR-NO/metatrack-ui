# AGENTS.md

This file gives coding agents project-specific instructions for working in this repository.

## Project Overview

MetaTrack GUI is the web frontend for the MetaTrack metadata management platform, built for life scientists in Norway through ELIXIR Norway. It lets users manage scientific projects, samples, assays, and related metadata.

- Production URL: `metatrack.no`
- API: `https://api.metatrack.no/api`
- Auth server: `https://auth.metatrack.no` using Keycloak

## Tech Stack

- React 19 + TypeScript with Vite
- TanStack Router with file-based routes and generated route tree
- TanStack React Query for server state
- shadcn/ui `new-york` style, Radix UI, Tailwind CSS v4
- React Hook Form + Zod v4 for forms
- TanStack React Table v8 for tables
- Recharts v3 and D3 for charts
- Keycloak 26 with OIDC + PKCE
- Sonner for notifications
- Motion for animations

## Package Manager And Commands

Use Bun only. `bun.lock` is present, so do not use npm or yarn for project commands.

```bash
bun dev
bun run build
bun run lint
bun run format:check
bun run format:write
bun run preview
```

`bun run build` runs `tsc -b` and then the Vite production build. There is currently no test framework configured.

## Project Structure

```text
src/
├── routes/           # TanStack Router file-based routes
├── components/       # Shared, dashboard, nav, card, and UI components
├── components/ui/    # shadcn/ui primitives
├── lib/              # API clients, config, Keycloak setup, shared types, utils
├── providers/        # Auth, Keycloak, and theme providers
├── hooks/            # Shared React hooks
├── content/          # Markdown content pages
├── routeTree.gen.ts  # Generated by TanStack Router; do not edit manually
└── main.tsx
```

## Code Style

- Use tabs for indentation with tab width 2, as configured in Prettier.
- Use double quotes and semicolons.
- Keep trailing commas in ES5-compatible positions.
- Let the Prettier Tailwind plugin sort Tailwind classes.
- Prefer TypeScript interfaces for component props, defined near the component.
- Use PascalCase for components and camelCase for functions, variables, and hooks.
- Keep strict TypeScript clean: no unused locals or parameters.
- Use the `@/*` path alias for imports from `src/*` when it improves clarity.

## Components And UI

- Use function components and hooks.
- Treat `src/components/ui/` as shadcn/ui primitives. Avoid manual edits unless fixing or intentionally customizing the primitive itself.
- Add shadcn components with:

```bash
bunx shadcn@latest add <component>
```

- Prefer composition around primitives over duplicating component logic.
- Use Tailwind CSS v4 utilities and CSS variables. Do not introduce CSS-in-JS.
- Use `class-variance-authority` for component variants when variants are reusable.

## Routing

- Routes live under `src/routes/`.
- `src/routeTree.gen.ts` is generated. Never edit it manually.
- Protected routes should use auth context and `useNavigate` consistently with existing dashboard routes.
- After adding or changing routes, expect TanStack Router tooling to regenerate the route tree during dev/build.

## API And Auth

- Use `src/lib/api-keycloak.ts` for authenticated API calls; it injects the Keycloak bearer token.
- Use `src/lib/api-client.ts` or `src/lib/auth-client.ts` only for existing public/basic-auth patterns.
- Read environment and endpoint configuration from `src/lib/config.ts` instead of hardcoding new URLs.
- Handle API errors with `try`/`catch` and surface user-facing failures with Sonner toasts when appropriate.
- For batch operations, prefer `Promise.allSettled` when partial failure needs to be handled.

## Forms And Validation

- Use React Hook Form for forms.
- Use Zod schemas for validation.
- Use `@hookform/resolvers/zod` for React Hook Form integration.
- Prefer the shared `src/components/form-field.tsx` helper where it matches the existing form pattern.

## State Management

- Use TanStack React Query for server state.
- Use React Context for auth and theme state.
- Use local `useState` for local UI state.
- Do not add Redux, Zustand, or another state library unless explicitly requested.

## Development Proxy

In development, Vite proxies:

- `/keycloak` to `https://auth.metatrack.no`
- `/api` to `https://api.metatrack.no`

Keep proxy behavior in `vite.config.ts` aligned with auth and API client expectations.

## Generated And External Files

- Do not edit `src/routeTree.gen.ts` manually.
- Do not reformat generated files or unrelated files.
- Do not edit files under `node_modules/`.
- Keep secrets, tokens, credentials, private keys, and local auth artifacts out of git.

## Agent Workflow

- Inspect the existing implementation before adding new patterns.
- Check `git status --short` before and after edits.
- Preserve user changes. Do not revert or overwrite work you did not make.
- Keep changes scoped to the user request and avoid broad refactors unless needed.
- Run the narrowest useful validation after edits, usually `bun run lint`, `bun run build`, or `bun run format:check` depending on the change.
- If validation cannot be run, state why and mention the command that should be run.
- When changing UI, verify desktop and mobile behavior where practical.
